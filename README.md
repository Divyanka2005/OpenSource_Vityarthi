Project Overview
This project is based on the study and analysis of Git as an open-source software. It includes understanding Git’s functionality, its role in version control, and practical implementation using shell scripts. The project demonstrates how Git is used in a Linux environment for managing code and collaboration.
Environment Setup
Step 1: Install Git
Download and install Git from the official website (git-scm.com).
After installation, verify it by running:
git --version
Step 2: Open Terminal
For Windows users, open Git Bash.
For Linux users, open the default terminal.
Dependencies
The project requires the following:
•	Bash Shell 
•	Basic Linux commands such as echo, read, date, and cat 
•	Git installed on the system 
Project Structure
The project contains the following files:
•	script 1 - System Identity Report  code 
•	 Script 2 — FOSS Package Inspector  code 
•	Script3 -Disk and Permission Auditor code 
•	Script4-Log File Analyzer code 
•	Script5-Open Source Manifesto Generator code 
•	Open source audit (project report)
•	README.md 
How to Run the Project
Step 1: Open terminal (Git Bash or Linux terminal)
Step 2: Navigate to the project folder using cd command
Step 3: Run any script using:
bash script_name.sh
Example:
bash script 1 - System Identity Report  code.sh

Scripts Description
•	script 1 - System Identity Report  code  - : This script displays system details along with Git installation information.
•	Script 2 — FOSS Package Inspector  code -: This script checks whether Git is installed and shows its version and details.
•	Script3 -Disk and Permission Auditor code -: This script analyzes directories and checks whether the current folder is a Git repository.
•	Script4-Log File Analyzer code -: This script shows commit history and analyzes Git logs.
•	Script5-Open Source Manifesto Generator code -: This script takes user input and generates a simple open-source manifesto file.

Configuration
•	Make sure Git is installed before running scripts 
•	Run git init in a folder before using Git-related scripts 
•	For log analyzer, ensure commits are available 
Output
All scripts display their output directly on the terminal. Screenshots of outputs are included in the report for verification.

Conclusion
This project demonstrates the use of Git as an open-source version control system and shows how shell scripting can interact with it. It highlights real-world applications of Git in software development and collaboration.
